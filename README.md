# AI Factory

AI Factory is a reproducible quantitative finance dataset factory with Python
reference engines and C++/CUDA production engines.

The active workflow builds model, product, and result databases with compact
JSON data, YAML specifications, same-basename generators, and validation
notebooks.

The registry has two tiers:

- `registry/validation` for benchmark databases, four-engine checks, tests, and
  notebooks;
- `registry/production` for large final databases generated by validated
  production engines, usually C++ CUDA.

`notebooks/validation/<model_family>` contains compact certification reports
grouped by model. Production notebooks are added only when a concrete
large-run audit requires one.

## Reproducibility

Result databases store one seed per result row. Result YAML files define the
shared simulation and pricing setup, including the engine, source files, Monte
Carlo path count, time-grid rule, outputs, and reproducibility scripts.

C++ CPU and C++ CUDA engines use the project Philox implementation and must
match to floating-point precision. Python CPU/GPU engines use PyTorch RNG and
serve as readable statistical benchmarks rather than pathwise references.
